# AI/Code Agent 使用记录

说明：本记录用于课程项目过程留痕。以下内容均按“AI 辅助建议与实现，人工选择、集成、测试、调参”的原则记录，不表述为 AI 独立完成项目。

## 后续记录强制项

后续每次重要修改都至少补齐以下信息，避免到答辩时只能回忆“改过”，却说不清“怎么定位、为什么这样改、人工做了什么”：

1. 问题现象
   例如：车辆无法响应 `WASD`、点击 `Play` 后黄车明显埋地、按 `2` 切车日志显示 `count=1`。
2. 人工先提出的怀疑原因
   例如：输入系统冲突、脚本挂载位置错误、场景里只有一辆车被注册、碰撞体参考面不一致。
3. 实际检查路径
   需要写清检查了哪些文件、哪些对象、哪些 Inspector 配置，例如：
   - `Assets/SimpleCarController.cs`
   - `Assets/VehicleManager.cs`
   - `Assets/Scenes/CollisionTestScene.unity`
   - `ARCADE - FREE Racing Car` prefab 的 `Rigidbody / BoxCollider / WheelCollider`
4. 真实定位结果
   必须写“最后确认的问题是什么”，不要只停留在“怀疑”。
5. 修改文件
   需要写出具体文件名。
6. 关键修改前/修改后
   需要说明逻辑变化，例如：
   - 修改前：`VehicleManager.vehicles` 场景里只保存了黄车
   - 修改后：运行时按车辆根对象自动补齐控制器并合并列表
7. 人工完成的工作
   例如：人工决定不切换到复杂整车方案、人工在 Unity 中重开场景、检查 Inspector、测试 `1/2` 切换、继续调参。
8. AI 辅助的工作
   例如：辅助比对 prefab、生成脚本初稿、整理 YAML 差异、补写文档。
9. 测试方式与结果
   至少写“怎么测、测到了什么、是否完全解决”。
10. 未解决问题
   必须保留，方便下一轮继续接力。
