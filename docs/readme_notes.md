## 当前 README 应表达的核心结论

README 应明确说明：

- 项目是基于 Unity 的车辆碰撞虚拟仿真系统。
- 当前最终场景为 `Assets/Scenes/CollisionTestScene.unity`。
- 当前场景中有 4 辆车，满足课程要求的 3-5 辆车。
- 4 辆车均可通过切换后驱动。
- 已实现车辆与车辆的物理碰撞。
- 已实现车辆与环境对象的物理碰撞。
- 已实现运行时 UI 参数面板和物理量可视化。
- 已整理 AI / Code Agent 使用记录。

## AI / Code Agent 使用说明口径

推荐表述：

> 本项目开发过程中使用 AI / Code Agent 辅助查找代码、分析问题、生成脚本修改、整理测试用例和文档。人工负责提出需求、观察 Unity 实际运行效果、判断问题是否解决、决定技术路线、集成代码、调试参数并最终验收。

可以举例：

- 车辆导入后材质粉紫，借助 Agent 排查材质和渲染管线问题。
- 初期车辆控制像开船，人工反馈后，Agent 辅助调整转向、阻尼、重心和 WheelCollider 驱动逻辑。
- 多车切换出现 `count=1 target=null`，Agent 辅助搜索场景 YAML 和 `VehicleManager`。
- 车辆低速顶车不明显，人工描述现象后，Agent 辅助修改低速碰撞和推车逻辑。
- UI 参数范围显示不正确，Agent 辅助扩展 `VehicleRuntimeUI`。
- Unity 崩溃时，Agent 辅助定位 PhysX 材质重复重挂问题。
