# 最终交付清单

说明：本文件用于给组员直接分发剩余工作，避免继续围着车辆底层物理细节反复调参。

## 必做项

1. 场景中补齐 `3-5` 辆车
2. 确认所有车辆都能驱动、切换、复位
3. 完成车车碰撞演示
4. 完成车辆与墙体、建筑、桥墩、树木、动态障碍物的碰撞演示
5. 保证参数面板可调、可见、单位清楚
6. 整理正式 `README.md`
7. 完成 PPT
8. 完成结题报告
9. 把 AI 使用过程写进 PPT 和结题报告

## 建议做

1. 给环境 primitive 替换更完整的材质包
2. 补充碰撞强度或最近碰撞对象显示
3. 补录测试截图或短视频
4. 给每个成员整理一页“自己做了什么”的材料

## 可以不深挖

1. 真实车损形变
2. 高保真轮胎动力学
3. 极端真实的低速贴车推行
4. 工业级悬挂、发动机、差速器细节

## 推荐验收标准

车车碰撞：
- 低速正面追尾时，目标车有可见位移
- 中速碰撞时，目标车有明显位移、偏转或姿态变化
- 高速碰撞时，能看到更明显的偏移、滑行或翻滚

环境碰撞：
- 墙体、建筑、桥墩、树木可以稳定阻挡车辆
- 箱子、路障、交通锥可以被撞动或撞倒

UI：
- 至少能调驱动力、阻尼、摩擦或质量中的一部分
- 至少能显示速度、质量、重力、车长估计、最近控制车辆等基础信息

## 最终展示建议

优先演示：
1. 多车切换与驾驶
2. 中低速车车碰撞
3. 高速侧撞或斜撞
4. 车辆撞墙体、建筑、桥墩、树木
5. 车辆撞动态箱子、路障、交通锥
6. UI 参数调节前后效果对比

不建议把大量时间花在：
- 继续反复深挖底层车辆动力学
- 为了极端真实感不断改坏已经稳定的功能

## 答辩口径建议

- 本项目以课程级简化仿真为目标，采用 Unity 6、`Rigidbody`、`WheelCollider` 和运行时参数面板构建车辆碰撞仿真系统。
- 重点完成了车辆驱动、车车碰撞、车辆与环境对象碰撞，以及物理参数的可视化与可调节。
- AI/Code Agent 用于辅助排查、生成脚本初稿、整理记录和测试清单；最终路线选择、参数取舍、集成与验收由人工完成。
