## 必须最终确认

1. 使用 Unity `6000.3.10f1` 打开项目。
2. 打开最终场景：

```text
Assets/Scenes/CollisionTestScene.unity
```

3. 点击 Play 后确认没有红色编译错误。
4. 确认 `Car1`、`Car2`、`Car3`、`Car4` 都能驾驶、切换、刹车、倒车、复位。
5. 确认车车碰撞有可见位移、偏转或姿态变化。
6. 确认车辆能撞环境对象：
   - 马路牙子
   - 人行道柱
   - 桥梁
   - 建筑边界
   - 城市道路边界
   - `CrashTestTree`
7. 确认 `CrashTestTree` 能被撞倒或推动。
8. 确认 `F2` UI 面板可打开，参数显示和调节可用。
9. 确认白天/黑夜切换可用。
10. 确认鼠标绕车和相机跟随可用。
11. 确认 README 中的车辆数量、按键和场景路径与实际一致。
12. 确认 PPT 和报告中写清楚第三方资源来源。
13. 确认 PPT 和报告中写清楚 AI / Code Agent 使用过程。

## 5 分钟演示清单

建议演示顺序：

1. 展示最终场景和 4 辆车。
2. 当前车辆前进、倒车、转向、刹车。
3. `1/2/3/4` 切换车辆。
4. 车辆与车辆碰撞。
5. 贴车持续推车。
6. 车辆撞路沿、人行道柱、桥梁或建筑。
7. 车辆撞 `CrashTestTree`。
8. `F2` 打开 UI 参数面板。
9. 展示速度、合力、质量、摩擦、驱动力等参数。
10. 展示白天/黑夜和鼠标绕车。

## 已知限制

答辩时可以主动说明：

- 本项目定位为课程级车辆碰撞虚拟仿真，不是工业级车辆动力学系统。
- 没有真实车损形变。
- 没有完整发动机、差速器、悬挂和高保真轮胎模型。
- 城市资源中的多数建筑、桥梁、路灯、路沿是静态碰撞体，不做整体可破坏。
- 额外配置的 `CrashTestTree` 用于满足植物碰撞和可推动对象演示。
