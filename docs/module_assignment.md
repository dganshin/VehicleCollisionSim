## 5 分钟演示路线

演示时间短时，建议只展示最稳、最能覆盖要求的功能：

1. 打开 `Assets/Scenes/CollisionTestScene.unity`
2. 展示场景中的 4 辆车和城市/高速环境
3. 驾驶当前车辆前进、倒车、转向、刹车
4. 用 `1/2/3/4` 切换车辆
5. 进行一次车车碰撞，最好是中低速追尾或侧撞
6. 持续顶住前车，展示推车效果
7. 撞击路沿、人行道柱、桥梁或建筑边界
8. 撞击 3 棵 `CrashTestTree` 中任意一棵
9. 按 `F2` 打开 UI 参数面板
10. 展示速度、合力、质量、驱动力、摩擦等参数
11. 展示白天/黑夜切换和鼠标绕车视角

## 答辩口径

推荐统一说法：

> 本项目使用 Unity 6、Rigidbody、WheelCollider 和运行时参数 UI 搭建车辆碰撞虚拟仿真系统。车辆模型和城市地图来自第三方免费资源，团队主要工作是资源导入与兼容、车辆控制逻辑、多车管理、车车碰撞、环境碰撞、可撞树、参数 UI、相机体验、测试流程和文档整理。AI / Code Agent 主要用于辅助查代码、生成脚本初稿、定位 Bug 和整理材料；每一轮实际效果都由人工在 Unity 中测试、反馈、取舍和集成。

## 不建议继续深挖的内容

- 真实车损形变
- 工业级发动机、差速器、悬挂模型
- 高保真轮胎动力学
- 建筑整体可破坏
- 光线追踪或复杂后处理

当前答辩重点应放在课程要求闭环：多车、可驱动、车车碰撞、环境碰撞、UI 参数可视化、AI 使用说明。
