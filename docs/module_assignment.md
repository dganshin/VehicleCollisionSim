## 当前建议的交付边界

必须完成：
- `3-5` 辆车都可驱动
- 车车碰撞可演示
- 车与环境对象碰撞可演示
- UI 能调车辆驱动与物理参数
- README、PPT、结题报告、AI 使用说明完整

建议完成：
- 环境材质包替换
- 碰撞对象名称和碰撞等级展示
- 统一录制测试截图或演示视频

可以不继续深挖：
- 真实车损形变
- 高保真轮胎动力学
- 极其真实的低速贴车推行

## 协作建议

- 所有成员都保留“人工操作证据”，例如 Inspector 调参截图、场景摆位截图、测试录屏、提交记录。
- 若使用 AI 生成脚本或文档，必须由对应模块负责人完成集成、复核和二次修改。
- PPT 和结题报告中必须写清楚：路线选择、脚本生成、调试建议可由 AI 辅助完成，但最终取舍、集成、测试、演示口径由人工决定。
- 演示阶段优先保证“课程级可信效果”，不要继续投入过多时间追求工业级真实车辆动力学。
